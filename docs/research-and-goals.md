# 图片文字整理工具研究与目标

## 背景

用户手机里保存了大量截图和图片，内容包括书摘、小启发、前端面试题、代码片段等。这些图片长期未整理，单靠相册浏览很难判断哪些值得保留、哪些可以删除。

## 产品目标

构建一个本地 Web 工具，把图片整理流程变成可批处理、可审阅、可删除的工作流：

`图片目录 -> 扫描入库 -> OCR 识别 -> 本地分析 -> 表格审阅 -> 保留/删除/导出`

核心界面是一个表格，每行对应一张图片，包含：

- 图片缩略图，支持点击放大查看原图。
- OCR 识别后的文字，支持编辑和复制。
- 本地分析结果，包括分类、摘要、价值评分、保留建议、过时风险、失真风险。
- 操作按钮，包括保留、复核、移入回收站。

## OCR 与 Ollama 取舍

OCR 主流程优先使用 PaddleOCR。原因：

- 它是专用 OCR 引擎，更适合批量处理中文、英文、截图和密集文本。
- 相比小型视觉大模型，在小字、长段落、截图排版上通常更稳定。
- 不依赖云端 API，符合本地优先目标。

如果 PaddleOCR 首次模型下载失败或初始化不可用，v1 使用 RapidOCR 作为轻量本地 fallback。Ollama 不作为 OCR 主引擎，而作为可选增强层：

- 输入 OCR 文本和可选图片。
- 输出结构化分析 JSON。
- 用于摘要、分类、价值判断、过时风险判断和失真风险判断。
- 如果 Ollama 未安装、模型缺失、响应超时或输出无法解析，自动回退规则分析。

首个推荐模型是 `qwen3.5:2b-q4_K_M`，它体积相对可控并支持图片输入，适合一般电脑先验证效果。

## 删除安全策略

删除操作不做永久删除。点击删除时：

- 弹出确认。
- 调用后端把原图移入 Windows 系统回收站。
- 数据库状态更新为 `trashed`。
- 如果移动失败，只记录错误并提示用户，不执行永久删除。

## v1 范围

- 本地 Web 应用。
- 扫描指定图片目录。
- SQLite 入库和去重。
- 生成缩略图。
- PaddleOCR 批量识别。
- 本地规则分析。
- 可选 Ollama 分析。
- 表格审阅界面。
- 原图预览。
- OCR 文本复制和编辑。
- 状态管理。
- 移入系统回收站。
- JSON/CSV 导出。

## v2 候选

- Notion/Obsidian 导出。
- Excel 格式化报告。
- 图片相似度聚类。
- 重复内容检测。
- 更强的知识标签系统。
- 接入云端视觉模型做高质量复核。
- 批量撤销回收站操作。
